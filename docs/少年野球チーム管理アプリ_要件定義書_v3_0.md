**プロジェクト名**: （仮称）Baseball Team Hub  
**作成日**: 2026年3月30日  
**最終更新日**: 2026年4月2日  
**バージョン**: 3.0  

> **v3.0 主な変更点（v2.0からの差分）**
> - 認証方式: LINE認証 + メール/パスワード認証の2方式に対応

## 1. プロジェクト概要

### 1.1 目的

少年野球チームに所属する選手（子供）とその保護者が、チーム運営に必要な情報共有・スケジュール管理・成績記録を一元的に行えるWEBアプリケーションを開発する。将来的にはマルチテナント化により複数チームが利用でき、チーム間の練習試合や合同練習のマッチングも可能にする。

### 1.2 技術スタック

| レイヤー | 技術 | 備考 |
|---------|------|------|
| フロントエンド | Next.js (App Router) + TypeScript | PWA対応でスマホからも利用 |
| バックエンド | Next.js API Routes / Server Actions | |
| データベース | Supabase (PostgreSQL) | team_idによるマルチテナント分離 |
| 認証 | LINE LIFF SDK + Supabase Auth | LINEログイン + メール/パスワード認証 |
| ファイルストレージ | Supabase Storage | 写真・スコアブック画像 |
| AI分析 | Claude API (Anthropic) | スコアブック画像の解析 |
| ホスティング | Vercel | Next.jsとの親和性 |
| 収益化 | アフィリエイト（楽天/Amazon/Yahoo等） | 買い物メニュー経由 |

### 1.3 開発方針

- 全機能を一括開発してからリリース
- まず自チーム1つで運用開始し、安定後に他チームへ展開
- team_idベースのマルチテナントで、同一DB内にデータを分離

### 1.4 対象ユーザーと利用規模（初期）

- チーム人数: 選手15〜25名 + 保護者30〜50名 + 指導者3〜5名
- 同時接続: 最大50名程度
- 将来的に地域リーグ全体（50チーム以上）への拡大を視野に入れる

## 2. ユーザーロールと権限

### 2.1 ロール設計

ロールは「役割（permission_group）」と「サイト管理者フラグ（is_admin）」の2軸で管理する。

#### サイト管理者（is_admin）
- チーム作成者に自動付与される管理者フラグ
- 役割に関係なく全機能にアクセス可能
- サイト管理者は他のユーザーを役割に関係なくサイト管理者に設定/解除できる
- 1チームに複数のサイト管理者が存在可能
- サイト管理者が0人になることは禁止（最後の1人は解除不可）
- 画面表示: 役割名の横に「⚙サイト管理者」バッジを表示

#### 役割一覧（permission_group）

| 役割名（画面表示） | permission_group | 説明 |
|------------------|-----------------|------|
| 監督 | director | チームの指導者。試合・練習の管理 |
| 会長 | president | チーム運営の代表者。会計・チーム間連携 |
| 副会長 | vice_president | 会長の補佐。参加承認・チーム間連携 |
| 部長 | captain | イベント管理・連絡投稿 |
| コーチ | coach | 指導者。試合・練習・成績管理 |
| 会計 | treasurer | 会費・請求・入金の管理 |
| 広報 | publicity | アルバム管理・写真投稿・連絡投稿 |
| 保護者 | parent | 自分の子供の情報閲覧・出欠登録 |

※ 旧バージョンの `system_admin`、`team_admin`、`manager`、`display_title` は廃止。役割名の日本語表示は `getRoleLabel()` ユーティリティで `permission_group` の値から変換する。

## 3. 機能要件

### 3.1 認証・ユーザー管理

#### F-AUTH-001: ログイン（2方式）
- **LINE認証**: LINE LIFF SDKを利用したソーシャルログイン。LINEプロフィール画像の自動取得
- **メール/パスワード認証**: Supabase Authのメール/パスワード認証。テスト用途やLINE未利用者向け
- ログイン画面にはメール/パスワード入力フォームとLINEログインボタンを両方配置
- ログイン状態の永続化（リフレッシュトークン）
- ログアウト機能（メニュー最下部に配置、Supabase Auth + LIFF両方のセッションをクリア）

#### F-AUTH-001b: 新規登録（メール/パスワード）
- `/register` 画面で表示名、メールアドレス、パスワード（8文字以上）、電話番号（任意）を入力
- Supabase Auth でユーザー作成 → users テーブルに保存
- 登録完了後はオンボーディング（チーム作成 or 招待参加）に遷移
- 招待リンク経由の場合は `redirect` パラメータで招待ページに戻る

#### F-AUTH-001c: LINE連携（後からの紐づけ）
- メール/パスワードで登録したユーザーが、マイページからLINEアカウントを連携可能
- LIFF認証 → LINEプロフィール取得 → users.line_user_id を更新
- 連携後はLINE通知が受け取れるようになる

#### F-AUTH-002: チーム新規作成（オンボーディング）
- 誰でもチームを新規作成可能（ログイン済みであれば）
- チーム作成時の入力項目: チーム名、活動地域、所属リーグ（任意）
- チーム作成者には自動的にサイト管理者（is_admin = true）が付与される
- 作成時に役割を選択（8役割から1つ）
- チーム作成後、招待リンクの生成画面に自動遷移
- **1ユーザー1チーム制限**: 既にチームに所属している場合はチーム作成不可

#### F-AUTH-003: 招待リンクによるメンバー招待
- チーム作成後、招待リンク（フルURL）とQRコードを自動生成
- 招待リンクには有効期限を設定可能（デフォルト7日、延長・再生成可能）
- 招待リンクをLINEグループや個別チャットで共有する想定
- 招待リンクからLINE認証またはメール/パスワード認証でログイン/新規登録が可能
- 招待参加者のデフォルト役割は `parent`（保護者）
- 招待リンクの利用回数制限（任意設定）
- **1ユーザー1チーム制限**: 既に別チームに所属している場合は参加不可（メッセージ表示）

#### F-AUTH-004: 保護者の参加フロー
1. 招待リンクからアプリにアクセス
2. LINEログインまたはメール/パスワードでログイン（未登録の場合は新規登録画面へ）
3. 保護者プロフィール入力: 氏名、電話番号
4. 子供（選手）情報の登録:
   - チームに既に選手が登録されている場合: チェックボックスで既存選手を選択し、関係（父/母/祖父/祖母/その他）を設定
   - 該当する子供がいない場合: 新規登録フォーム（氏名、背番号（任意）、学年（1〜6年）、ポジション、投打）
   - 1人以上の子供を選択または登録が必須
5. チームへの参加が完了（承認不要 or 設定で選択可能）
6. ホーム画面に遷移

#### F-AUTH-005: 保護者と選手の紐づけ（多対多）
- 保護者アカウントに対して1人以上の選手（子供）を紐づけ
- 1人の選手に複数の保護者を紐づけ可能（父・母・祖父母）
- 1人の保護者に複数の選手を紐づけ可能（兄弟）
- 既に登録済みの選手との紐づけ: チェックボックスで選択（重複登録を防止）
- 後からマイページで子供の追加・情報編集が可能
- **保護者間の子供情報自動共有**: 保護者が子供を追加した場合、同じ子供を共有している他の保護者にも自動的に紐づけが追加される

#### F-AUTH-006: ロール管理
- 初期権限: チーム作成者はサイト管理者（is_admin = true）+ 選択した役割、招待参加者は `parent`
- サイト管理者がメンバー管理画面から役割を変更可能
- サイト管理者は他のユーザーをサイト管理者に設定/解除可能
- サイト管理者が0人になることは禁止（最後の1人は解除不可）

#### F-AUTH-007: チーム所属制限
- **1ユーザー1チーム制限**: 1ユーザーは1つのチームにのみ所属可能
- 別のチームに参加するには、先に現在のチームを退会する必要がある
- 退会機能: マイページから「チームを退会する」ボタンで退会可能（確認ダイアログ付き）
- サイト管理者が最後の1人の場合は退会不可

### 3.2 スケジュール・カレンダー

#### F-CAL-001: イベント登録
- イベント種別: 練習、練習試合、公式戦、合同練習、その他（懇親会、総会等）
- 入力項目: 日時（開始・終了）、場所（自由入力 + Google Maps連携）、集合時間、集合場所、持ち物、備考
- 繰り返しイベント対応（毎週土曜の練習など）
- イベント作成時にLINE通知を送信

#### F-CAL-002: カレンダー表示
- 月表示・週表示・リスト表示の切り替え
- イベント種別ごとの色分け
- 過去のイベントはグレーアウト表示

#### F-CAL-003: 出欠管理（選手）
- 各イベントに対して選手ごとの出欠を登録: 参加 / 不参加 / 未定
- 保護者が自分の子供の出欠を登録
- 出欠の締切日時を設定可能
- 未回答者へのリマインド通知（LINE）
- 出欠一覧表示（管理者・コーチ向け）

#### F-CAL-004: 出欠管理（保護者）
- 保護者自身の参加・不参加を登録
- 当番制の場合の当番割り当て表示
- 車出し可否・乗車可能人数の登録

### 3.3 連絡・コミュニケーション

#### F-COM-001: 連絡投稿
- 監督・コーチ・会長・会計・部長が投稿可能
- テキスト + 画像（複数枚）の投稿
- 重要度フラグ（通常 / 重要 / 緊急）
- カテゴリタグ（練習、試合、事務連絡、会計、その他）
- 投稿時にLINE通知を送信（重要度に応じた通知レベル）

#### F-COM-002: リアクション・コメント
- いいね（既読確認を兼ねる）
- コメント（テキストのみ）
- いいね数・コメント数の表示
- 未読の連絡にバッジ表示

#### F-COM-003: 既読管理
- 連絡ごとに既読・未読を管理
- 管理者向け: 誰が既読済みかの一覧表示
- 未読者へのリマインド送信機能

---

### 3.4 スコアブック・試合記録

#### F-SCR-001: スコアブック画像登録
- 試合ごとにスコアブックの写真を複数枚アップロード
- 画像は試合イベントに紐づけ
- アップロード上限: 1試合あたり20枚まで
- 対応フォーマット: JPEG, PNG, HEIC

#### F-SCR-002: スコアブックAI解析
- アップロードされたスコアブック画像をClaude APIで解析
- 解析結果: 打席結果（安打、三振、四球、犠打、エラー等）、打点、得点、盗塁
- 解析結果の手動修正機能（AI解析の誤りを人間が修正）
- 解析済みデータの確定フロー（コーチ以上が承認）

#### F-SCR-003: 試合結果登録
- スコア（イニングごとの得点）
- 先発メンバー・打順・守備位置
- 投手成績（投球回、被安打、奪三振、四死球、失点、自責点）
- 対戦チーム情報

---

### 3.5 選手成績・成長記録

#### F-STAT-001: 身体測定記録
- 記録項目: 身長(cm)、体重(kg)、測定日
- 入力頻度: 月1回程度（任意）
- グラフ表示: 身長・体重の推移（折れ線グラフ）
- 平均身長・体重との比較表示（学年別）

#### F-STAT-002: 体力測定記録
- ベースラン（塁間走）タイム（秒）
- 遠投距離（メートル）
- その他の計測項目を自由に追加可能（50m走等）
- 測定日ごとの記録
- 推移グラフ表示

#### F-STAT-003: 打撃成績
- スコアブック解析データから自動集計
- 集計指標: 打率、出塁率、長打率、OPS、打点、得点、盗塁数、三振率、四球率
- 通算成績・月別成績・大会別成績の切り替え表示
- 成績推移グラフ（月別打率の変動など）

#### F-STAT-004: 守備・投手成績
- 守備: 刺殺、補殺、失策、守備率
- 投手: 防御率、勝敗、投球回、奪三振率、WHIP
- ポジション別の出場イニング数

#### F-STAT-005: 成長ダッシュボード
- 選手個人ページに全指標をまとめて表示
- レーダーチャートでバランス表示（打撃力、走力、守備力、体力）
- 前月比・前年比の成長率表示
- 保護者は自分の子供のみ閲覧可、指導者は全選手閲覧可

### 3.6 ランキング

#### F-RANK-001: チーム内ランキング
- ランキング対象指標（設定で有効・無効を切替可能）:
  - 打率、本塁打数、打点、盗塁数、出塁率、OPS
  - ベースランタイム、遠投距離
- 期間指定: 通算 / 今月 / 今シーズン / 大会別
- 最低打席数・登板数の足切り条件を設定可能
- 上位3名のハイライト表示

#### F-RANK-002: ランキング表示設定
- 監督が表示するランキング項目を選択可能
- 子供のモチベーションに配慮し、非表示にする項目も設定可能
- ランキング更新通知（週次）

### 3.7 キッズ機能（選手向け成長・モチベーション機能）

#### F-KIDS-001: マイ選手カード
- プロ野球カード風のデザインで選手の個人ステータスを表示
- 表示項目: 顔写真（アルバムから選択 or アップロード）、背番号、ポジション、学年、投打
- ステータス表示: 打率、出塁率、盗塁数、遠投距離、ベースランタイムなどの主要指標
- レーダーチャート（打撃力・走力・守備力・体力の4〜6軸）
- カードランク: 成績・バッジ数に応じて自動でランクアップ
  - ブロンズ（初期）→ シルバー → ゴールド → プラチナ
  - ランクアップ条件はシステム管理者が設定（例: バッジ5個でシルバー、打率.300以上でゴールド等）
- カードデザインはランクに応じて装飾が変化（枠の色・背景パターン）
- 試合・測定データが更新されると自動的にカードのステータスも更新
- カード画像をダウンロード・LINE共有可能
- 保護者が自分の子供のカードを閲覧、指導者は全選手のカードを閲覧可能

#### F-KIDS-002: 成長タイムライン
- 選手の「はじめて」や記録的な瞬間を自動検出してタイムラインに表示
- 自動検出するマイルストーン:
  - はじめてのヒット、はじめての打点、はじめてのホームラン
  - はじめての三振（投手として）、はじめての完投
  - 打率の自己ベスト更新、遠投の自己ベスト更新、ベースランの自己ベスト更新
  - 身長が○cm超え（10cm単位の節目）
  - 入団○年目の記念日
- 各マイルストーンに日付・試合情報・数値を自動付与
- 監督・コーチ・保護者がマイルストーンにお祝いコメントを追加可能
- タイムライン表示: 入団日から時系列で縦スクロール
- 保護者が手動でマイルストーンを追加可能（「はじめて自主練した日」等）

#### F-KIDS-003: 今月のふりかえり
- 月末に自動生成される1ヶ月のサマリーレポート
- 表示内容:
  - 練習参加回数（全体の何%に参加したか）
  - 試合数・打席数
  - 打率の変化（先月比 ↑↓）
  - 身長・体重の変化
  - 獲得したバッジ
  - 今月のベストプレー（最多安打の試合など）
- ポジティブなメッセージを自動生成（例: 「先月より打率が5分上がったよ！」「練習皆勤賞だね！」）
- ネガティブな表現は使わない（成績が下がった場合は「来月はもっと打てるようになろう！」等）
- 保護者・選手のホーム画面に月初に通知
- 過去のふりかえりも月別にアーカイブ閲覧可能

#### F-KIDS-004: 今週のMVP・がんばったで賞
- 監督・コーチ（team_admin）が毎週1名を選んで表彰
- 表彰カテゴリ:
  - 今週のMVP（成績ベース）
  - がんばったで賞（態度・努力ベース）
  - ナイスプレー賞（特定のプレーに対して）
- 表彰時の入力: 選手名、カテゴリ、コメント（理由）
- 成績だけでなく「声出しがよかった」「後輩に優しく教えていた」「最後まであきらめなかった」など態度面も表彰対象
- 表彰はホーム画面のお知らせ欄 + LINEグループに通知
- 全員に機会が回るよう、未受賞者を管理画面で提案表示（「○○くんは最近表彰されていません」）
- 表彰履歴は選手の成長タイムラインにも自動追加
- 過去の表彰一覧をアーカイブとして閲覧可能

#### F-KIDS-005: バッジ・称号システム
- 特定の条件を達成すると自動的にバッジを獲得
- プリセットバッジ（システムで用意）:
  - 皆勤賞（月間練習全参加）
  - 初ヒット、初打点、初ホームラン、初盗塁
  - 初三振（投手）、初完投、初勝利
  - 連続出塁（3試合以上連続で出塁）
  - 守備の鉄人（10試合連続ノーエラー）
  - 俊足ランナー（ベースラン自己ベスト更新）
  - 遠投キング（遠投自己ベスト更新）
  - 努力の星（目標チャレンジ達成時）
- 監督がカスタムバッジを作成可能（バッジ名、条件の説明、アイコン色を指定）
- 獲得したバッジはマイ選手カードに表示（最大6個を選択してカードに配置）
- バッジ獲得時にLINEグループに通知（任意設定）
- バッジ図鑑: 全バッジの一覧と獲得条件を閲覧。未獲得バッジはシルエット表示

#### F-KIDS-006: マイ目標チャレンジ
- 選手（または保護者が代理で）が個人目標を設定
- 目標の入力: 目標タイトル、目標値、期限（任意）
- 目標の例:
  - 「遠投30mを超える」
  - 「打率2割5分以上」
  - 「ベースラン5秒台を出す」
  - 「10試合連続ノーエラー」
- 進捗の自動トラッキング:
  - 測定・試合データが登録されるたびに進捗バーが自動更新
  - 達成率を%で表示（例: 遠投目標30m、現在27m → 90%）
- 目標達成時: 「努力の星」バッジを自動付与、成長タイムラインに記録
- 監督・コーチが目標に対して応援コメントを投稿可能
- 同時に設定できる目標は最大3つ
- 達成済み・期限切れの目標はアーカイブに移動

#### F-KIDS-007: チームチャレンジ
- 監督（team_admin）がチーム全体の共通目標を設定
- チームチャレンジの例:
  - 「チーム通算100安打達成」
  - 「全員で練習参加100回」
  - 「チーム打率2割5分以上」
  - 「今シーズン10勝」
- 進捗はホーム画面に常時表示（プログレスバー + 現在値/目標値）
- 全選手の貢献を合算して進捗を自動計算
- 達成時: ホーム画面で祝賀表示 + LINEグループに通知
- 同時に設定できるチームチャレンジは最大2つ
- 達成履歴をアーカイブ表示

#### F-KIDS-008: チームメンバー図鑑
- チームメイト全員の選手カードを図鑑形式で一覧表示
- 表示項目: 選手カード（F-KIDS-001）のコンパクト版
- ポジション別・背番号順・学年別のフィルター・ソート
- 各選手のプロフィール追加情報（任意入力）:
  - 好きなプロ野球選手
  - 好きなプレー・得意なプレー
  - 目標・将来の夢
- プロフィールは保護者が編集可能
- 指導者は全選手、保護者は自チームの選手のみ閲覧可

#### F-KIDS-009: ベストプレー集
- 試合のスコアデータから「今日のベストプレー」を自動ピックアップ
- ピックアップ条件:
  - 猛打賞（3打数3安打以上）
  - 決勝打（試合を決めた打点）
  - 完封・無失点投球
  - 1試合複数盗塁
  - ファインプレー（監督が手動で追加）
- 試合終了後（スコア確定後）に自動生成
- 表示形式: 試合ごとのベストプレーカード
  - 選手名、プレー内容、該当する写真（アルバムから手動で紐づけ可能）
- ベストプレーはホーム画面のお知らせ + LINEグループに通知（任意）
- ベストプレー集を時系列で閲覧可能（チーム全体のハイライト集）
- 選手個人の成長タイムラインにも自動追加

### 3.9 会計・集金管理

#### F-ACC-001: 会費設定・請求作成
- 月会費の金額設定（チーム全体で統一）
- 月会費の自動請求生成（毎月1日に自動作成、日付は設定変更可能）
- 都度集金の請求作成: 大会参加費、遠征費、備品購入費、合宿費など
- 請求項目: 件名、金額、対象者（全員 / 個別選択）、支払期限、備考
- 兄弟割引などの個別金額調整に対応
- 請求作成時にLINEグループへ通知

#### F-ACC-002: 入金記録・消込
- 保護者ごとの入金記録（日付、金額、支払方法）
- 支払方法: 現金、銀行振込、その他
- 請求に対する消込（一括消込 / 個別消込）
- 過払い・不足金の管理
- 入金確認時に保護者へ個別LINE通知（任意設定）

#### F-ACC-003: 収支台帳（入出金管理）
- チーム全体の入出金を時系列で記録
- 収入カテゴリ: 月会費、大会参加費、遠征費、寄付、その他
- 支出カテゴリ: グラウンド使用料、備品購入、大会登録費、交通費、保険料、その他
- カテゴリは自由に追加可能
- 月別・年度別の収支サマリー表示
- 残高の自動計算・表示
- レシート・領収書の画像添付機能

#### F-ACC-004: 支払い状況一覧・未納管理
- 保護者ごとの支払い状況一覧（月別マトリクス表示）
- ステータス: 未請求 / 請求済 / 入金済 / 未納（期限超過）
- 未納者の一覧表示とフィルタリング
- 未納リマインド通知（LINEグループ経由 or 個別LINE通知）
- リマインド送信履歴の記録
- 保護者自身が自分の支払い状況を確認できる画面
- 年度ごとの支払い履歴

### 3.10 チーム間連携（マルチテナント対外機能）

#### F-INTER-001: チーム公開プロフィール
- チーム名、活動地域、所属リーグ、活動日、ホームグラウンド
- チームロゴ・写真
- 連絡先（監督 or 会長のLINE表示名 or メール）
- 公開・非公開の切り替え

#### F-INTER-002: チーム検索
- 活動地域・所属リーグでの絞り込み検索
- チーム名のキーワード検索
- 検索結果からチームプロフィール閲覧

#### F-INTER-003: チーム間メッセージ
- 監督・会長ロールのユーザー同士がメッセージ送受信
- テキスト + 日程候補の送信
- 既読表示
- チーム単位のスレッド管理（チームAとチームBの会話履歴）

#### F-INTER-004: 練習試合・合同練習の申し込み
- 申し込みフォーム: 希望日時（複数候補）、場所、形式（練習試合/合同練習）、備考
- 申し込み → 承認 → 確定のワークフロー
- 確定した試合は両チームのカレンダーに自動登録
- キャンセル・日程変更フロー

#### F-INTER-005: 対戦成績の共有
- 練習試合の結果を両チームで共有
- 対戦成績の履歴表示（チームA vs チームB: ○勝○敗）
- 各試合のスコア・詳細へのリンク

### 3.11 通知・LINEグループ連携

#### F-NOTIF-001: LINEグループ通知連携
- チーム設定画面でLINE公式アカウント（Bot）をチームのLINEグループに追加
- LINE Messaging API（Bot）を利用し、アプリ内の投稿をLINEグループに自動転送
- 通知対象のアクション:
  - 連絡投稿（重要度に応じてメッセージの装飾を変える）
  - イベント作成・変更・キャンセル
  - 出欠リマインド（締切前日）
  - 会費請求・未納リマインド
  - チーム間メッセージ受信（監督・会長に個別通知）
  - 練習試合申し込み・承認・確定
- LINEメッセージにアプリ内該当画面へのディープリンクを含める
- LINEグループ内での通知フォーマット:
  - 投稿者名 + カテゴリアイコン + 本文プレビュー（100文字まで）+ 詳細リンク
  - 画像付き投稿の場合はサムネイル1枚を含める
- 通知頻度の制御: まとめ通知モード（1時間以内の投稿をまとめて1通知にする）設定可

#### F-NOTIF-002: 個別LINE通知
- ユーザー個別へのLINE通知（Botからの1:1メッセージ）
- 対象: 入金確認、未納リマインド、出欠未回答リマインドなど個人宛の内容
- ユーザーごとに個別通知のON/OFF設定可能

#### F-NOTIF-003: アプリ内通知
- ベルアイコンに未読件数バッジ
- 通知一覧画面（時系列）
- 通知タップで該当画面へ遷移

### 3.12 買い物（ショップ）・アフィリエイト

#### F-SHOP-001: 商品管理（システム管理者専用）
- システム管理者のみが商品の登録・編集・削除・表示順変更を行える
- 商品登録項目:
  - 商品名、説明文、商品画像（複数枚）
  - カテゴリ（バット、グローブ、スパイク、ユニフォーム、練習用品、備品、その他）
  - カテゴリはシステム管理者が自由に追加可能
  - 対象学年（低学年向け / 高学年向け / 全学年 など）
  - 推奨コメント（「試合用におすすめ」「練習用に最適」など）
  - 販売リンク（複数ECサイト対応）:
    - リンクURL（アフィリエイトタグ付き）
    - ECサイト名（楽天 / Amazon / Yahoo / その他自由入力）
    - 参考価格（表示用、リアルタイム連動はしない）
  - 公開 / 非公開フラグ
  - おすすめ度（星1〜5）
  - 表示優先度（並び替え用）

#### F-SHOP-002: 商品一覧・詳細表示
- ナビゲーションメニュー内に「買い物」メニューを配置
- 商品一覧はカテゴリ別にタブまたはフィルターで切り替え
- 商品カード表示: 商品画像、商品名、推奨コメント、参考価格、おすすめ度
- 商品詳細画面: 全画像、説明文、各ECサイトへの購入ボタン
- 購入ボタンタップで外部ECサイトに遷移（アフィリエイトリンク経由）
- 全チームのユーザーに共通の商品一覧を表示（チーム個別の商品設定は将来対応）

#### F-SHOP-003: チームおすすめ（ピン留め + コメント）
- チーム管理者（team_admin）・副会長（vice_president）が商品をピン留め可能
- ピン留め時におすすめコメントを入力（必須）
  - 例:「うちのチームはこのバットを使っています。低学年にちょうど良い重さです」
  - 例:「来月の大会までにスパイクを買い替えてください」
- コメントは投稿者名（表示呼称付き）+ 投稿日時とともに表示
- ピン留めされた商品は「買い物」メニュー内で「チームのおすすめ」セクションとして最上部に表示
- コメントの編集・削除はコメント投稿者本人またはteam_adminが可能
- 1商品に対して複数のチーム管理者がそれぞれコメント可能

#### F-SHOP-004: おすすめ商品のお知らせ表示
- チーム管理者がピン留め+コメントした商品のみ、ホーム画面のお知らせ欄に表示
- お知らせの表示形式:
  - 「[監督] ○○さんがおすすめ商品を紹介しました」
  - 商品画像サムネイル + 商品名 + コメントプレビュー（50文字まで）
  - タップで商品詳細画面へ遷移
- システム管理者が登録しただけの商品（ピン留めなし）はお知らせに表示されない
- お知らせ表示は自チームの管理者がコメントしたものだけに限定
- LINEグループへの通知もピン留め+コメント時のみ送信（任意設定）
  - 通知内容: 投稿者名 + 商品名 + コメント抜粋 + アプリへのリンク

#### F-SHOP-005: チーム備品・定期購入おすすめ
- システム管理者が「チーム備品」カテゴリで定期的に購入が必要な商品を登録
- 例: チーム共通のボール、バッティングティー、練習ネット、救急用品
- 「定期購入おすすめ」バッジを付与可能

#### F-SHOP-006: 将来的なEC機能への拡張準備
- 商品テーブルは将来的な自前販売（在庫管理・決済・配送）に対応できる拡張性を持たせる
- 現段階では以下はスコープ外とし、将来対応として設計だけ考慮:
  - オンライン決済（Stripe / PayPay等）
  - 在庫数管理
  - 注文管理・配送ステータス
  - 卸仕入れ価格と販売価格の管理
  - チーム単位での一括注文

## 5. セキュリティ要件

### 5.1 データアクセス制御（Row Level Security）

- Supabase RLS（Row Level Security）を全テーブルに適用
- 全データアクセスにteam_idによるフィルタリングを強制
- 保護者は自分の子供のデータのみ閲覧可（user_children経由）
- 指導者ロールは同一チーム内の全選手データを閲覧可
- チーム間メッセージは送信元・送信先チームのメンバーのみ閲覧可
- 会計データ（invoices, payments, ledger_entries）は会計・会長・監督ロールのみ書き込み可
- 保護者は自分宛の請求・入金データのみ閲覧可
- LINE Botトークンは暗号化してDB保存し、サーバーサイドでのみ復号
- 商品データ（shop_products等）の書き込みはシステム管理者のみ（RLSで強制）

### 5.2 個人情報保護

- 子供の写真はチームメンバーのみ閲覧可（外部公開不可）
- チーム公開プロフィールには個人を特定できる情報を含めない
- 退団時のデータ保持ポリシー: 成績データは匿名化して保持、個人情報は削除
- プライバシーポリシーの掲示と同意取得

### 5.3 画像セキュリティ

- Supabase Storageのアクセスポリシーでチーム外アクセスを防止
- アップロード時のファイル検証（MIMEタイプ・ファイルサイズ）
- メタデータ（EXIF）の自動削除（位置情報漏洩防止）

## 6. 非機能要件

### 6.1 パフォーマンス

- ページ読み込み: 3秒以内（3G回線でも）
- 画像アップロード: 1枚あたり10秒以内
- AI解析: 1試合分のスコアブック解析を60秒以内
- カレンダー表示: 1秒以内

### 6.2 可用性

- 稼働率: 99.5%以上
- 計画メンテナンス: 平日深夜に実施

### 6.3 スケーラビリティ

- 初期: 1チーム（50ユーザー程度）
- 中期: 10チーム（500ユーザー程度）
- 長期: 50+チーム（2,500+ユーザー程度）
- Supabase無料枠 → 有料プランへの移行タイミングを見極め

### 6.4 対応デバイス・ブラウザ

- スマートフォン（iOS Safari, Android Chrome）を最優先
- PWA対応でホーム画面追加を推奨
- PC（Chrome, Safari, Edge）でも利用可能
- レスポンシブデザイン

### 6.5 多言語対応

- 初期は日本語のみ
- 将来的な多言語対応を考慮した設計（i18nキー管理）

## 9. 開発時の技術的な注意事項

### 9.1 マルチテナント設計

- 全テーブルにteam_idカラムを持たせ、RLSポリシーで強制フィルタリング
- チーム間データは専用テーブル（inter_team_messages, match_requests）で管理
- インデックス: team_idを含む複合インデックスを主要テーブルに設定
- テナント間のデータ漏洩を防ぐためのRLSポリシーのテスト必須

### 9.2 スコアブックAI解析のフロー

```
画像アップロード → Supabase Storage保存
  → Claude API（Vision）でOCR + 構造化解析
  → JSON形式で解析結果を返却
  → フロントで解析結果をプレビュー表示
  → ユーザーが手動修正
  → コーチ以上が確定承認
  → player_game_statsテーブルに保存
  → 打率等の集計指標を再計算
```

### 9.3 LINE連携の構成

#### 認証（LIFF）
```
ユーザー → LIFF App (Next.js) → LIFF SDK初期化
  → LINE認証 → アクセストークン取得
  → Supabase Auth (カスタムトークン)
  → セッション確立
```

#### LINEグループ通知（Bot）
```
アプリ内アクション（投稿・イベント作成・請求作成等）
  → Server Action / API Route でイベント検知
  → team_line_config からグループID・トークン取得
  → batch_mode の判定
    → OFF: 即時送信（LINE Messaging API Push Message）
    → ON: キューに蓄積 → batch_interval_min 後にまとめ送信
  → LINEグループにメッセージ配信
  → メッセージ内にアプリへのディープリンク（LIFF URL）を含める

個別通知（入金確認・未納リマインド等）
  → users.line_user_id を使って1:1 Push Message
```

#### LINE Bot セットアップ手順（チーム管理者向け）
```
1. LINE Developers でMessaging API チャネルを作成
2. チャネルアクセストークンを発行
3. アプリのLINE連携設定画面でトークンを登録
4. LINE Botをチームのグループに招待
5. Webhook URLをアプリのエンドポイントに設定
6. グループ参加時にgroup_idを自動取得・保存
```

### 9.4 推奨ライブラリ

| 用途 | ライブラリ | 理由 |
|------|-----------|------|
| UIコンポーネント | shadcn/ui | カスタマイズ性が高い |
| カレンダー | FullCalendar or react-big-calendar | 実績あり |
| グラフ | Recharts | React親和性が高い |
| フォーム | React Hook Form + Zod | バリデーション込み |
| 状態管理 | Zustand or TanStack Query | 軽量 |
| 画像処理 | Sharp (サーバーサイド) | リサイズ・EXIF削除 |
